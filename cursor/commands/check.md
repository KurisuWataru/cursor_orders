# /check

実装した変更内容を総合的にレビューするコマンドです。

## 指示

以下の手順を順番に実行してください。

### 1. 変更内容の収集

まず、レビュー対象の変更内容を収集してください。

- `git status` を実行し、変更されたファイルの一覧を取得する
- `git diff`（またはステージ済みの場合は `git diff --staged`）を実行し、差分を取得する
- `git log --oneline -10` を実行し、直近のコミット履歴を確認する
- 変更されたファイルにリンターエラーがないか確認する

### 2. レビューの実行

`requirement-reviewer` サブエージェントを起動し、以下の情報を渡してレビューを実行させてください。

- **要件**: このセッションの最初の方で提示された要件、または指定した Issue/ファイルの内容
- **差分**: 手順1で取得した変更差分
- **関連ファイル**: 変更に関連するファイル（変更ファイルが参照・依存しているファイル）

### 3. 結果の報告と次のアクション

レビュー結果に基づいて、以下のように対応してください。

- **合格の場合**: コミットメッセージの提案を表示し、コミットするか確認する
- **要修正の場合**: 指摘事項を重大度順に一覧表示し、修正案を具体的に提示する
- **不合格の場合**: Critical な指摘事項を優先的に表示し、修正すべき箇所と方針を明確に提示する
